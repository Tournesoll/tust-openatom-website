# 🔧 打包后应用故障排除

## 问题：点击 exe 后页面打不开

### 可能的原因

1. **服务器启动失败** - Node.js 路径问题
2. **端口被占用** - 8888 端口已被使用
3. **路径问题** - 打包后的路径结构不同
4. **配置文件问题** - config.json 路径错误

### 调试步骤

1. **查看控制台输出**
   - 右键点击任务栏的应用图标
   - 选择"打开开发者工具"（如果有）
   - 或者查看控制台窗口的输出

2. **检查端口占用**
   ```powershell
   netstat -ano | findstr :8888
   ```

3. **手动测试服务器**
   - 在打包后的目录中找到 `resources/app/server.js`
   - 尝试手动运行（需要 Node.js）

### 已修复的问题

✅ **Node.js 路径问题** - 已修复，打包后使用 Electron 的 Node.js
✅ **路径解析问题** - 已修复，正确处理打包后的路径结构

### 如果问题仍然存在

请提供以下信息：
1. 控制台输出的错误信息
2. 是否有窗口弹出（即使空白）
3. 任务管理器中是否有相关进程
